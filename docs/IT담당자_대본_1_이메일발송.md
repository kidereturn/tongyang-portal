# IT 담당자 대본 ①: 이메일 발송 기능 협조 요청

## 핵심 요청 (1줄)
> "내부회계 포털에서 **결재 알림 메일**과 **관리자 공지 메일**을 자동 발송할 수 있도록, **회사 그룹웨어 메일 서버(SMTP)** 사용 권한을 발급해주세요."

---

## 통화 / 미팅용 대본 (3분 분량)

안녕하세요, 내부회계팀 [본인 이름] 입니다. 잠시 시간 괜찮으세요?

저희가 운영 중인 **내부회계 포털 (`tyia.vercel.app`)** 에서 두 가지 자동 이메일 발송이 필요한 상황입니다.

### 시나리오 1 — 결재 알림 메일
- 담당자가 증빙을 결재상신하면 → **승인자에게 자동으로 안내 메일** 발송
- 메일 내용: "OO 통제활동에 대한 증빙이 상신되었습니다. 아래 링크에서 검토해주세요."
- 메일 링크 클릭 시: 포털 자동 로그인 후 해당 결재 화면으로 이동

### 시나리오 2 — 관리자 일괄 공지
- 관리자가 포털에서 공지사항을 등록하면 → **전 임직원 이메일로 발송**
- 예: "이번 분기 증빙 마감일 안내", "내부회계 교육 안내" 등

---

## IT 담당자에게 부탁드릴 사항 (4가지)

### 1. SMTP 발송 권한 발급
- 그룹웨어 메일 서버 (예: `mail.tongyanginc.co.kr`) 의 SMTP 발송 계정 1개 필요
- 발신자 주소: `noreply@tongyanginc.co.kr` 또는 `accounting@tongyanginc.co.kr`
- 인증 방식: **SMTP AUTH (TLS)** 권장. 또는 IP 화이트리스트
- 발송 한도: 일 최대 500건 (476명 + 여유)

### 2. SPF / DKIM / DMARC 설정
- 외부에서 발송된 메일이 **스팸 차단되지 않도록** DNS 레코드 협조
- 도메인 관리자가 DNS 에 다음 추가:
  - `SPF`: `v=spf1 include:vercel.com include:[그룹웨어 SMTP 도메인] ~all`
  - `DKIM`: 그룹웨어 측에서 제공하는 공개키 등록
  - `DMARC`: `v=DMARC1; p=quarantine; rua=mailto:postmaster@tongyanginc.co.kr`

### 3. 그룹웨어 SSO 연동 (선택)
- 메일 링크 클릭 시 자동 로그인 → **이미 그룹웨어 로그인된 상태면 그대로 통과**
- 옵션 A: SAML / OIDC SSO 연동 (개발 1주)
- 옵션 B: **Magic Link** — 메일에 일회용 토큰 URL 포함, 클릭 시 즉시 로그인 (개발 2일, 추천)

### 4. 보안 검토
- 발송 트래픽 로그 보관: 최소 30일
- 발신자 사칭 방지: From 헤더 = SPF 인증 도메인과 일치
- 첨부 파일 없음 (안전)

## 기대 효과
- 결재 누락 방지 (담당자→승인자 평균 응답 시간 30% 단축 예상)
- 임직원 공지 도달률 100% (게시판만 사용 시 약 60%)
- 그룹웨어 사용 흐름과 통합 → 별도 시스템 학습 부담 0

---

## 제 측 준비 완료 사항
- 메일 템플릿 (한글 HTML) 5종 준비됨
- 발송 큐 (Supabase Edge Function `send-approval-email`) 구현 완료 — SMTP 정보만 넣으면 즉시 동작
- 발송 실패 시 재시도 로직 (3회 exponential backoff) 적용
