작성일: 2026년 4월 · 사이트 URL: https://tyia.vercel.app · 담당: 내부회계팀

## 📧 대본 1 — 그룹웨어 IT 담당자 협의 (이메일 발송 기능)

### 협의 목적
내부회계관리 포털에서 다음 두 가지 이메일 발송 기능을 구현하기 위해 그룹웨어(또는 회사 메일 서버) 발송 권한을 받고자 합니다.

### 요청 기능 2가지

**① 결재상신 알림 메일 (자동)**
- 트리거: 담당자가 증빙을 결재상신하면, 해당 활동의 승인자에게 자동 메일
- 메일 내용: "[내부회계 결재] 통제번호 / 활동명 / 담당자명 / 포털 바로가기 링크"
- 빈도: 사용자 액션마다 1건 (예상 일평균 5–20건)
- 회신 주소: noreply 또는 내부회계팀 공용 메일

**② 관리자 → 전 사용자 공지 메일 (수동)**
- 트리거: 관리자가 포털에서 작성·발송 (월 1–2회 추정)
- 수신자: 활성 사용자 476명 (사번 기준 자동 추출)
- 메일 내용: 관리자가 입력한 제목·본문 + HTML 포맷
- 빈도: 월 1–2회

### IT 담당자에게 요청할 사항 (체크리스트)

1. **SMTP 발송 정보 제공**
   - SMTP 서버 호스트 / 포트 / 보안(TLS·STARTTLS)
   - 인증 계정(아이디·비밀번호 또는 토큰)
   - 발송 가능 도메인 (예: `noreply@tongyanginc.co.kr`)

2. **일일/시간당 발송 한도 확인**
   - 단건 알림 (예상 일 20건 미만)
   - 일괄 메일 (예상 회당 476건, 월 2회) → **SMTP 한도 초과 우려 시 대안 협의**

3. **스팸 분류 방지를 위한 SPF/DKIM 설정 협조**
   - 발송 도메인 SPF 레코드에 포털 발송 IP 또는 SMTP 서버 추가
   - 가능하면 DKIM 서명 키 발급

4. **테스트 환경 협의**
   - 검증용 메일 5–10건을 IT 담당자 본인 계정으로 발송하여 정상 도착 확인 후 본 운영 시작

### 한 줄 요약 (구두 협의용)
> "내부회계 포털에서 ① 결재 알림 자동 메일 ② 관리자 공지 일괄 메일 발송 기능을 구현하려고 합니다. SMTP 인증 정보와 일일 한도를 확인해 주시고, `noreply@tongyanginc.co.kr` 발송 도메인 SPF 등록을 요청드립니다. 본 운영 전 테스트 5건 함께 검증하면 좋겠습니다."

## 🔒 대본 2 — 보안 안내 (직원 정보 보호 설명)

### 핵심 메시지 (3가지)

**① 모든 데이터는 암호화 전송 + 저장 (HTTPS + at-rest 암호화)**

- 사이트는 100% HTTPS 로만 동작 (HTTP 접속 시 자동 리다이렉트)
- HSTS (HTTP Strict Transport Security) 설정으로 브라우저가 항상 HTTPS 강제
- DB(Supabase Postgres) 와 Storage 의 데이터는 모두 AWS KMS 기반 at-rest 암호화 (AES-256)
- 비밀번호는 bcrypt 단방향 해시 — 관리자도 평문 비밀번호를 볼 수 없음

**② Row Level Security(RLS) 로 권한 분리 — 본인 데이터만 접근 가능**

- 담당자(owner): 자기가 담당하는 통제활동 / 자기 증빙 / 자기 알림만 SELECT 가능
- 승인자(controller): 자기가 승인 책임인 활동만 접근
- 관리자(admin): 전체 데이터 접근 가능
- **DB 레벨 정책**으로 강제 — 클라이언트 코드 우회 불가능 (URL 조작·API 직접 호출 모두 차단)
- 75개의 RLS 정책 적용 (`anon`, `authenticated`, `authenticator` 역할별)

**③ 추가 보안 장치**

- HIBP (HaveIBeenPwned) 연동으로 유출된 비밀번호 사용 차단
- 보안 헤더 적용:
  - `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload`
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: SAMEORIGIN` (clickjacking 방지)
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=()`
- 증빙 파일 다운로드는 1시간 만료 signed URL 사용 (영구 공개 링크 없음)
- 감사 로그(login_logs) 영구 보관 — 모든 접속 기록 추적 가능

### 자주 받을 질문 답변

| 질문 | 답변 |
|---|---|
| "비밀번호가 새어나갈 수 있나요?" | 비밀번호는 bcrypt 로 해시되어 DB 에 저장됩니다. 관리자도 원본 비밀번호를 볼 수 없으며, 비밀번호 재설정 시에만 새 해시로 덮어씁니다. |
| "내가 올린 증빙을 다른 사람이 보나요?" | 본인의 활동을 담당하는 승인자(1명)와 관리자(내부회계팀)만 볼 수 있습니다. 같은 부서 동료라도 권한이 없으면 접근 불가입니다. |
| "회사 외부에서 누가 해킹할 수 있나요?" | HTTPS + RLS + 보안 헤더 + 1시간 만료 다운로드 링크의 다단계 보호로 외부 공격은 매우 어렵습니다. 또한 모든 접속 기록이 남아 추적 가능합니다. |
| "데이터가 갑자기 사라지면?" | Supabase 는 매일 자동 백업되며, 7일치 시점복구(PITR)가 가능합니다. 오류로 삭제된 데이터도 복구 요청으로 되돌릴 수 있습니다. |

### 한 줄 요약 (구두 안내용)
> "이 포털은 모든 데이터가 암호화되고, 권한이 없는 사람은 접근조차 못 하도록 데이터베이스 레벨에서 차단됩니다. 비밀번호는 관리자도 볼 수 없고, 다운로드 링크도 1시간이면 만료됩니다."

### 슬라이드 4: 담당자 — 증빙 업로드 + 결재상신 (3분)

```
[화면 캡처 4컷]
1. 증빙관리 목록 — 본인 담당 활동 표시
2. 활동 우측 [업로드] 버튼 클릭
3. 모달 — 모집단별 파일 첨부
4. 모든 파일 채워지면 [결재상신] 활성화
```

**5단계 흐름**:
1. **증빙관리 → 본인 활동 찾기**
2. 활동 우측 **[업로드]** 버튼 클릭
3. 모달이 열리면 **각 모집단(거래)별로 PDF/Excel 파일 첨부**
   - 첨부 즉시 자동 중간저장 ✅
   - 잘못 올렸으면 [삭제] 또는 [교체] 가능
4. 모든 파일 첨부되면 **[결재상신]** 버튼이 파란색으로 활성화
5. 클릭 → 승인자에게 자동 메일 발송 (구현 후) → 승인 대기

> **TIP**: 결재상신 후 관리자가 **수정제출** 요청 시 메모를 남기며, 담당자가 다시 업로드 후 재상신할 수 있습니다.

### 슬라이드 5: 승인자 — 승인/반려 (1.5분)

```
[화면 캡처]
증빙관리에서 본인이 승인자인 활동 표시
액션 컬럼에 [승인] / [반려] 버튼

[클릭 후]
승인 → 즉시 '승인완료' 상태
반려 → 사유 입력 → 담당자에게 통보
```

**승인자 액션**:
1. **증빙관리** 페이지에서 본인이 승인자인 활동 확인
2. 담당자가 결재상신한 활동에 **[승인]** / **[반려]** 버튼 노출
3. **[확인]** 버튼으로 모달 열어 첨부된 모든 증빙을 한 번에 검토
   - 단일 다운로드: 파일별 [다운로드] 버튼
   - 일괄 다운로드: [전체 ZIP 다운로드] 버튼
4. 검토 후 [승인] 또는 [반려] 클릭
   - 반려 시 **사유 입력** 필수 → 담당자에게 자동 통보

### 슬라이드 6: 관리자 — 검토상태 + 메모 (1분)

```
[화면 캡처]
관리자만 보이는 검토결과 컬럼
```

**관리자 검토 흐름**:
1. 증빙관리 → 활동별 검토결과 컬럼
2. **[미검토 → 검토중 → 완료]** 단계별 변경
3. 문제 있는 활동은 **[수정제출]** 선택 + 메모 입력
   - 자동으로 모든 승인 취소 + 담당자 재작업 상태로 복원
4. **[저장]** 클릭 → 즉시 반영
5. 메모는 담당자/승인자에게도 표시 (검토 의견 공유)
